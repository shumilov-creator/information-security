# Information Security Utilities

Это репозиторий с настольным приложением Windows (один файл `ЗИ.cpp`). Ниже — краткие шаги, чтобы быстро забрать свежий код и вставить его в Visual Studio.

## Быстрая синхронизация в Visual Studio
1. Откройте встроенный терминал Visual Studio или привычный Git Bash в корне проекта.
2. Получите последние изменения: `git pull`.
3. Если у вас уже открыт проект, выполните «Reload» в Solution Explorer, либо просто закройте и откройте `.vcxproj`/решение заново, чтобы подтянуть обновлённые файлы.
4. При необходимости обновите зависимости через «Project → Retarget Projects» (если изменилась версия SDK) и соберите проект.
5. Для локальных правок используйте стандартный цикл: `git status` → `git add` → `git commit -m "Сообщение"` → `git push`, после чего можно открыть Pull Request на GitHub.

Эти шаги позволяют быстро синхронизировать код и продолжить работу без ручного копирования файлов.

## Если `git pull` ругается на трекинг ветки
В текущем репозитории не настроен удалённый origin, поэтому `git pull` выдаст сообщение о том, что нет tracking-ветки. Добавьте удалённый и настройте отслеживание один раз:

```bash
git remote add origin <URL вашего репозитория>
git branch --set-upstream-to=origin/work work
```

После этого команды `git pull` и `git push` будут работать как обычно. Если основная ветка называется иначе, замените `work` на актуальное имя.

## Если терминал пишет «fatal: not a git repository»
1. Убедитесь, что открыта папка самого репозитория. В проводнике/VS Code должно быть видно скрытую папку `.git` рядом с `README.md` и `ЗИ.cpp`.
2. В терминале перейдите в корень репозитория: `cd /путь/к/репозиторию` и проверьте, что команда `git status` работает без ошибок.
3. Если вы случайно открыли подкаталог, вернитесь на уровень выше или переоткройте правильную папку через **File → Open Folder…**.
4. Если `.git` отсутствует (папка была скопирована без истории), клонируйте репозиторий заново `git clone <URL>` или выполните `git init`, добавьте remote и настройте трекинг ветки, как описано выше.

## Если терминал пишет «git не распознано» в Windows
1. Установите [Git for Windows](https://git-scm.com/download/win) (или через Microsoft Store: «Git» от Git Project). При установке оставьте параметр *Add Git to PATH* включённым.
2. Перезапустите VS Code после установки, чтобы подтянулись новые переменные окружения.
3. В терминале VS Code проверьте, что Git доступен: `git --version`. Если команда работает, можно выполнять `git pull`/`git push`.
4. Если ошибка сохраняется, убедитесь, что VS Code открывает правильную оболочку (PowerShell/Command Prompt). Дополнительно можно вручную добавить путь `C:\\Program Files\\Git\\bin` (или `C:\\Program Files\\Git\\cmd`) в переменную окружения PATH через *System Properties → Advanced → Environment Variables*.
